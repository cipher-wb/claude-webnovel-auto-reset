# Claude 自动重启工具使用说明

## 这是什么工具

这是一个给 `Claude Code + webnovel-writer` 用的自动化小工具。

它的作用只有一件事：

你在某一本小说目录里启动这个工具后，它会在每一轮新的 Claude 会话里自动输入、自动选中、自动确认 `/webnovel-write`。  
等这一章真正写完后，它会自动判断这章是不是**真的写完了**。  
如果确认已经写完，它会**再额外等 1 分钟**，然后在**同一个 Terminal 标签页**里重启 Claude，让上下文清空，方便你继续写下一章。

它会在每次拉起新的 Claude 会话后，自动输入：

```text
/webnovel-write
```

它只负责：

1. 监听当前章节是否成功完成
2. 成功后等待 1 分钟
3. 重启当前 Claude 会话
4. 在新会话里自动发送、自动选中、自动确认 `/webnovel-write`

如果你的项目里本来就有一个还没彻底结束的旧章节任务，这个工具也会把它当成当前要继续监听的任务。

这个工具在 macOS 上依赖两类能力：

1. `Terminal.app` 正常运行 Claude
2. `System Events` 代替你自动按键

所以第一次使用前，如果系统弹出权限提示，你要点允许。

## 你的小说目录长什么样

你真正要进入的，不是工具目录，而是**某一本小说自己的根目录**。

一个正常的小说项目根目录，至少应该长这样：


你最需要确认的是：

```text
.webnovel/state.json
```

只要当前目录里能看到这个文件，通常就说明你现在站在了正确的小说根目录。

## 小白怎么找对目录

如果你完全不熟悉命令行，就按下面一步一步来。

### 方法 1：在 Finder 里先找到小说目录

1. 打开 Finder
2. 找到你那本小说的文件夹
3. 确认这个文件夹里面有：
   - `.webnovel`
   - `正文`
   - `大纲`
   - `设定集`

如果你看不到 `.webnovel`：

1. 在 Finder 里按 `Command + Shift + .`
2. 这样会显示隐藏文件夹
3. 再看看有没有 `.webnovel`

### 方法 2：把文件夹拖进 Terminal

如果你已经打开了 Terminal：

1. 先输入：

```zsh
cd 
```

注意：`cd` 后面留一个空格。

2. 然后把 Finder 里的小说文件夹，直接拖进 Terminal 窗口
3. Terminal 会自动帮你补全路径
4. 按回车

### 方法 3：确认你现在在哪个目录

输入：

```zsh
pwd
```

它会显示你当前所在目录。

再输入：

```zsh
ls -la
```

如果输出里能看到下面这些目录或文件：

```text
.webnovel
正文
大纲
设定集
```

就说明你大概率已经进入正确目录了。

你还可以进一步确认：

```zsh
ls -la .webnovel/state.json
```

如果这个命令能显示文件信息，说明当前目录就是正确的小说根目录。

## 第一次使用怎么操作

下面是最简单、最稳的用法。

### 第 1 步：打开 Terminal

打开 macOS 自带的 Terminal.app。

### 第 1.5 步：如果系统提示权限，请点允许

这个工具会自动帮你在 Claude 里输入 `/webnovel-write`。

macOS 可能会弹出和下面类似的权限请求：

- 允许 Terminal 控制你的电脑
- 允许 System Events 辅助功能

如果弹了，就点允许。

如果你之前点过“不允许”，可以手动去这里打开：

```text
系统设置 -> 隐私与安全性 -> 辅助功能
```

通常你需要确认这些项目有权限：

- `Terminal`
- `System Events`

### 第 2 步：进入你的小说目录

例如：

```zsh
cd /Users/你的用户名/你的小说目录
```

如果你不会手打路径，就用上面说的“把文件夹拖进 Terminal”的方式。

### 第 3 步：确认目录对不对

输入：

```zsh
ls -la
```

你应该能看到：

```text
.webnovel
正文
大纲
设定集
```

### 第 4 步：运行自动重启工具

输入：

```zsh
zsh /Users/cipher/AI/连续写作方案测试/webnovel_auto_reset.zsh
```

运行成功后，它会先拉起一个新的 Claude 会话。

如果你想限制循环次数，比如只自动跑 3 轮成功章节，就输入：

```zsh
zsh /Users/cipher/AI/连续写作方案测试/webnovel_auto_reset.zsh 3
```

这里的 `3` 表示：

- 成功完成 3 章后自动停止
- 不会再自动拉起第 4 轮 Claude
- 如果中途失败，还是会提前停止

### 第 5 步：等待脚本自动输入命令

脚本拉起 Claude 后，会自动输入并确认：

```text
/webnovel-write
```

你不需要自己再手打这条命令。

如果 Claude 弹出斜杠菜单，这是正常现象。工具会自动完成下面 3 步：

1. 输入 `/webnovel-write`
2. 选中菜单里的 `webnovel-write`
3. 再确认执行

如果你看到一行类似下面的内容，也属于正常现象：

```text
/webnovel-writer:webnovel-write
```

这表示工具已经把命令选中了，接下来会继续自动确认执行。

如果当前这个 skill 会继续追问你章号或下一步信息，你再按它的提示继续输入即可。

### 第 6 步：章节完成后等待 1 分钟

如果这章真的成功完成了：

1. 工具会先检测到“完成信号”
2. 再额外等待 1 分钟
3. 然后自动重启当前 Claude
4. 重启后就是一个全新的 Claude 会话
5. 新会话会自动再次输入、选中并确认 `/webnovel-write`

## 平时怎么用

日常用法其实就一句话：

1. 先进入某一本小说目录
2. 运行自动脚本
3. 等脚本自动执行 `/webnovel-write`
4. 等写完
5. 等 1 分钟自动重启
6. 新会话会自动再次执行 `/webnovel-write`

你不用关窗口，也不用新开 Terminal。  
整个过程都在**同一个 Terminal 标签页**里完成。

如果你想自己指定循环次数，规则很简单：

```zsh
zsh /Users/cipher/AI/连续写作方案测试/webnovel_auto_reset.zsh
```

这表示：

- 不限次数
- 会一直循环，直到你手动停止

```zsh
zsh /Users/cipher/AI/连续写作方案测试/webnovel_auto_reset.zsh 5
```

这表示：

- 最多只自动跑 5 轮成功章节
- 第 5 轮写完并完成 1 分钟等待后，脚本自动结束

注意：

- 这里的次数，是“成功完成的章节数”
- 不是 Claude 重启次数
- 如果某一章失败了，脚本不会继续硬跑到目标次数，而是直接停下来

## 怎么停止

这个工具提供两种停止方式。

### 方式 1：温和停止

先看脚本启动时打印出来的 `wrapper PID`。

然后在另一个 Terminal 窗口里输入：

```zsh
touch /tmp/webnovel_auto_reset.<wrapper_pid>.stop
```

例如：

```zsh
touch /tmp/webnovel_auto_reset.12345.stop
```

作用：

- 告诉工具“不要再自动重启了”
- 如果当前还在等待重启，它会停止自动化
- 这是比较温和、推荐的停止方式

### 方式 2：强制停止

在另一个 Terminal 窗口里输入：

```zsh
kill <wrapper_pid>
```

例如：

```zsh
kill 12345
```

作用：

- 直接结束 wrapper
- 属于更强硬的停止方式
- 一般在工具卡住或你想立刻停掉时使用

简单理解：

- `touch /tmp/webnovel_auto_reset.<wrapper_pid>.stop`：推荐，温和停止
- `kill <wrapper_pid>`：强制停止

## 它到底是怎么运转的

这一段是原理说明。你不想看可以跳过，但如果以后排查问题，会非常有用。

### 第 1 层：wrapper 负责启动 Claude

文件：

```text
/Users/cipher/AI/连续写作方案测试/webnovel_auto_reset.zsh
```

它负责：

1. 检查你当前目录是不是小说根目录
2. 启动当前这个 Terminal 标签页里的 `claude`
3. 同时启动一个后台 watcher
4. 等 watcher 判断是否需要重启
5. 在每一轮新 Claude 会话里自动发送 `/webnovel-write`

它本身**不负责判断章节是否完成**。

### 第 2 层：watcher 负责监听完成信号

文件：

```text
/Users/cipher/AI/连续写作方案测试/webnovel_completion_watcher.py
```

它只盯当前小说目录下面这些内容：

```text
.webnovel/workflow_state.json
.webnovel/observability/call_trace.jsonl
.webnovel/state.json
.webnovel/index.db
.webnovel/summaries/chNNNN.md
正文/第NNNN章*.md
```

### 第 3 层：它怎么判断“这一章写完了”

它不是只看正文文件有没有生成。

它会分 3 层判断：

#### 第一层：看调用事件日志

它会监听：

```text
.webnovel/observability/call_trace.jsonl
```

如果看到：

```text
task_completed
```

而且这个事件对应的是：

```text
webnovel-write
```

就说明主流程已经报告“这章完成了”。

#### 第二层：看工作流状态文件

它会继续看：

```text
.webnovel/workflow_state.json
```

如果这里面的最新历史记录显示：

- 命令是 `webnovel-write`
- 章号正确
- 状态是 `completed`
- `current_task` 已经变成 `null`

那也说明这章已经完整完成。

#### 第三层：看业务产物有没有全部落地

因为这个仓库的工作流记录是 best-effort，不保证每次都 100% 写成功。  
所以工具还会做兜底检查。

它会检查下面这些东西是不是都生成了：

- `正文/第NNNN章*.md`
- `.webnovel/state.json`
- `.webnovel/index.db`
- `.webnovel/summaries/chNNNN.md`

而且这些文件必须是**这次写作之后新生成或新更新的**。

只有这样，它才会把这章当成“真的写完”。

### 第 4 层：为什么还要再等 1 分钟

这是保底机制。

原因很简单：

- 有时刚写完时，最后的日志、状态、落盘还在收尾
- 有时你也可能想在最后 1 分钟里再看看当前会话

所以工具不会一检测到成功就立刻踢掉 Claude。  
它会再多等 60 秒。

### 第 5 层：1 分钟到了以后会发生什么

1. watcher 通知 wrapper：现在可以重启了
2. wrapper 只关闭**它自己启动的这个 Claude 进程**
3. 不会去全局杀别的终端里的 Claude
4. 然后在**同一个 Terminal 标签页**里重新拉起一个新的 Claude
5. 新 Claude 拉起后，wrapper 会自动输入 `/webnovel-write`

结果就是：

- Terminal 窗口不变
- 标签页不变
- Claude 会话变成全新的
- 上一章的上下文被清空

## 哪些情况不会自动重启

下面这些情况，工具都不会自动帮你重启：

### 1. 当前这轮根本没有成功进入 `/webnovel-write`

比如你只运行了：

```text
/webnovel-plan 1
```

或者：

```text
/webnovel-review 1-5
```

这些命令都不会触发自动重启。

### 2. 本章失败了

如果 `webnovel-write` 失败了，工具不会帮你自动清空上下文。  
这样你还可以留在当前会话里看错误、排查问题。

### 3. 你在成功完成前手动退出了 Claude

如果这章还没完成，你就自己 `/exit` 了，那工具会认为这是人工中断，不会自动重启。

### 4. 你所在目录不对

如果你不是在小说根目录里运行这个工具，而是在别的目录里运行，它会直接报错退出。

## 常见报错和排查

### 报错 1：当前目录不是小说根目录

典型原因：

- 你在工具目录里运行了脚本
- 你在 workspace 上层目录里运行了脚本
- 你没进入具体某一本小说的目录

排查命令：

```zsh
pwd
ls -la
ls -la .webnovel/state.json
```

如果最后一个命令报错，就说明目录不对。

### 报错 2：`.webnovel` 不存在

说明这个目录还不是一个已初始化的 webnovel 项目目录。

你需要先确认：

- 这是不是那本小说真正的根目录
- 这本小说是否做过 `/webnovel-init`

### 报错 3：一直不重启

先检查有没有这些文件：

```zsh
ls -la .webnovel/workflow_state.json
ls -la .webnovel/observability/call_trace.jsonl
ls -la .webnovel/state.json
ls -la .webnovel/index.db
ls -la .webnovel/summaries
ls -la 正文
```

如果这些文件都正常存在，但还是一直不重启，通常有 4 种可能：

1. 这章其实还没真正跑完
2. 这章失败了
3. 当前这轮并没有真正跑起来 `/webnovel-write`
4. 工作流记录缺失，且兜底产物也没有完全落盘

### 报错 4：找不到 `wrapper PID`

脚本启动时会打印类似：

```text
wrapper PID: 12345
```

如果你忘了记，可以在另一个终端里用：

```zsh
ps aux | grep webnovel_auto_reset.zsh
```

来查进程号。
